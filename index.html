What is Ralph Vibe?

Ralph Vibe automates the Ralph Wiggum technique for autonomous AI development. Instead of manually crafting prompts and managing context for each coding session, Ralph Vibe provides the infrastructure through a simple command-line interface.

The Ralph Wiggum technique addresses a fundamental challenge in AI-assisted development: maintaining reliable, consistent output across long development sessions. By structuring work into discrete iterations with fresh context, specifications as the source of truth, and test-driven completion, Ralph Vibe enables truly autonomous development workflows.

With Ralph Vibe, you describe what you want to build, and the tool handles the rest—generating specs, planning tasks, and executing work in a structured loop until your project is complete.

Quick Start

Prerequisites

  • Deno runtime installed
  • Claude Code (requires Pro or Team subscription)

Installation

deno install -g -A -n ralph jsr:@anthropic/ralph

Basic Usage

The quickest way to start is with the --vibe flag, which chains all commands automatically:

ralph init --vibe

This will initialize your project, conduct an interview to gather requirements, generate an implementation plan, and begin the autonomous work loop.

Commands Reference

ralph init
Sets up project scaffolding with specs directory, AGENTS.md, and build instructions. Creates the foundation for autonomous development.
ralph start
Conducts an interactive interview to gather project requirements. Generates detailed specifications based on your responses.
ralph plan
Analyzes your specifications and generates an ordered list of implementation tasks in IMPLEMENTATION_PLAN.md.
ralph work
Executes the autonomous build loop. Picks up tasks from the plan, implements them, validates with tests, and iterates until complete.
ralph spec
Adds new features to an existing project via an interview process. Great for extending functionality after initial setup.
--vibe flag
Chains commands automatically: init → start → plan → work. The fastest path from zero to working project.

Philosophy

Ralph Vibe is built on five reliability principles that make autonomous development work:

  1. Fresh context each iteration

    Every work iteration starts with clean context. No accumulated confusion, no stale assumptions—just the current task and relevant specifications.

  2. Specs as truth

    Written specifications are the single source of truth. Claude follows what's documented, not what might have been discussed or implied.

  3. Single-task focus

    One feature per iteration. This constraint prevents scope creep and ensures each change is complete and tested before moving on.

  4. Test-driven completion

    Tasks aren't done until tests pass. This provides an objective completion criteria that doesn't rely on subjective assessment.

  5. File-based persistence

    All state lives in files—specs, plans, code. Context can reset, but progress persists through the filesystem.
